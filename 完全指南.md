## 第一章 认识 Claude Code

### 1.1 什么是 Claude Code？

Claude Code 是 Anthropic 开发的**命令行 AI 编程助手**。它可以直接在你的终端中运行，能够：

- 📖 阅读和理解你的代码项目
- ✏️ 编辑、创建、删除文件
- 🖥️ 执行终端命令
- 🔍 搜索代码、分析逻辑
- 🐛 定位和修复 Bug
- 📊 生成报告和文档

### 1.2 三种运行模式

| 模式 | 命令 | 适用场景 |
|------|------|------|
| **交互式会话** | `claude` | 多轮对话，复杂任务 |
| **单次问答** | `claude -p "问题"` | 快速咨询，脚本中调用 |
| **管道模式** | `echo "..." \| claude -p -` | 与其他工具组合 |

### 1.3 与其他工具的区别

| 工具 | 定位 | 交互方式 |
|------|------|------|
| Claude Code | 命令行 AI 编程助手 | 终端原生，可读写文件、执行命令 |
| GitHub Copilot | IDE 代码补全 | VS Code / JetBrains 内嵌 |
| Cursor | AI 编辑器 | 完整 IDE，图形界面 |
| ChatGPT | 通用对话 | Web / App |

> Claude Code 的优势：**终端原生，可直接操作文件系统和执行命令，无需复制粘贴代码。**

## 第二章 环境准备：安装 nvm 和 Node.js

Claude Code 基于 Node.js 运行，所以首先需要安装 Node.js。推荐使用 nvm（Node Version Manager）来管理 Node.js 版本。

### 2.1 什么是 nvm？

nvm 可以让你在同一台机器上安装和切换多个 Node.js 版本，避免版本冲突。就像 conda 管理 Python 环境一样。

### 2.2 安装 nvm

**WSL / Linux：**

```bash
# 第 1 步：下载并安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash

# 第 2 步：让 nvm 在当前终端生效
source ~/.bashrc

# 第 3 步：验证安装
nvm --version
# 应该输出类似 0.40.3 的版本号
```

**macOS：**

```bash
# 第 1 步：用 Homebrew 安装
brew install nvm

# 第 2 步：配置 shell
mkdir ~/.nvm
echo 'export NVM_DIR="$HOME/.nvm"' >> ~/.zshrc
echo '[ -s "$(brew --prefix)/opt/nvm/nvm.sh" ] && \. "$(brew --prefix)/opt/nvm/nvm.sh"' >> ~/.zshrc
source ~/.zshrc

# 第 3 步：验证
nvm --version
```

**Windows：**

Windows 不支持 nvm 的原生 Linux 版本。你需要：

```powershell
# 方式 1：安装 WSL2（推荐）
wsl --install -d Ubuntu
# 然后进入 WSL，按 Linux 流程操作

# 方式 2：使用 nvm-windows
# 访问 https://github.com/coreybutler/nvm-windows/releases
# 下载 nvm-setup.exe 并安装
```

### 2.3 安装 Node.js

```bash
# 第 1 步：安装最新的 LTS（长期支持）版本
nvm install --lts

# 第 2 步：切换到 LTS 版本
nvm use --lts

# 第 3 步：验证
node --version
# 应该输出 v22.x.x 或 v24.x.x

npm --version
# 应该输出 10.x.x 或更高
```

### 2.4 注意事项

- ⚠️ Claude Code 要求 Node.js ≥ 18
- ⚠️ 推荐使用 LTS 版本（更稳定）
- ⚠️ 不要用 `sudo npm install -g`，nvm 管理的版本不需要 sudo

## 第三章 安装 Claude Code

### 3.1 方式 A：npm 全局安装（推荐）

```bash
# 第 1 步：安装
npm install -g @anthropic-ai/claude-code

# 第 2 步：验证
claude --version
# 应该输出类似 2.1.177 (Claude Code) 的信息
```

### 3.2 方式 B：npx 不安装直接运行

如果你只想试用，不想全局安装：

```bash
npx @anthropic-ai/claude-code
```

每次运行都会临时下载，适合一次性使用。

### 3.3 方式 C：更新到最新版本

```bash
npm update -g @anthropic-ai/claude-code
claude --version
```

### 3.4 常见安装错误与解决

| 错误信息 | 原因 | 解决方法 |
|----------|------|----------|
| `claude: command not found` | npm 全局 bin 路径不在 PATH | 见下方解决方案 |
| `EACCES: permission denied` | 权限不足 | 不要用 sudo；用 nvm 管理 Node.js |
| `npm ERR! code E404` | npm registry 问题 | `npm config set registry https://registry.npmjs.org/` |
| `Node.js version must be >= 18` | Node 版本太旧 | `nvm install --lts && nvm use --lts` |

**解决 `command not found`：**

```bash
# 查看 npm 全局安装路径
npm config get prefix

# 将全局 bin 目录加入 PATH
echo 'export PATH="$HOME/.nvm/versions/node/$(node -v)/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# 再次验证
claude --version
```

## 第四章 认证配置

Claude Code 需要连接到 AI 模型才能工作。有两种方式：

- **Anthropic 官方 API**（适合海外用户，需要国外信用卡）
- **MiniMax 第三方 API**（适合国内用户，支持支付宝/微信）

### 4.1 配置目录

```bash
mkdir -p ~/.claude
```

### 4.2 MiniMax API 配置（国内用户首选）

> ⚠️ **开始前必读**：不同 MiniMax 环境使用不同的 API 地址和认证字段。若下述配置不生效，请先阅读 [4.3 节 API 变体说明](#43-️-minimax-api-地址与认证字段变体说明)。

MiniMax 是国内 AI 平台，提供兼容 Anthropic API 的接口，国内网络直连。

```bash
cat > ~/.claude/settings.json << 'EOF'
{
  "model": "MiniMax-M2.7",
  "permissionMode": "acceptEdits",
  "env": {
    "ANTHROPIC_BASE_URL": "https://api.minimax.chat/anthropic",
    "ANTHROPIC_API_KEY": "sk-你的MiniMax-API-Key",
    "API_TIMEOUT_MS": "600000",
    "DISABLE_NONESSENTIAL_TRAFFIC": "1",
    "CLAUDE_CODE_DISABLE_TERMINAL_TITLE": "1"
  },
  "autoCompact": {
    "enabled": true,
    "maxTokens": 512000
  }
}
EOF
```

> 🔑 **如何获取 MiniMax API Key？**  
> 访问 https://platform.minimax.chat 注册账号 → 控制台 → API Keys → 创建密钥。

### 4.3 ⚠️ MiniMax API 地址与认证字段变体说明

> **重要**：不同 MiniMax 平台环境可能使用不同的 API 地址和认证字段名：
>
> | 使用场景 | API 地址 | 认证字段名 |
> |----------|----------|------------|
> | MiniMax 海外版 | `https://api.minimax.chat/anthropic` | `ANTHROPIC_API_KEY` |
> | MiniMax 国内版 | `https://api.minimaxi.com/anthropic` | `ANTHROPIC_AUTH_TOKEN` |
>
> **如果 `ANTHROPIC_API_KEY` 认证失败（401 Unauthorized），请将字段名改为 `ANTHROPIC_AUTH_TOKEN`。**
> **请以 MiniMax 平台的最新文档为准**，登录后查看 API 面板中的实际地址和 Key 格式。

### 4.4 Anthropic 官方 API（海外用户）

```bash
cat > ~/.claude/settings.json << 'EOF'
{
  "model": "claude-sonnet-4-20250514",
  "apiKeyHelper": "claude-keys"
}
EOF

# 存储 API Key 到系统 keychain
echo "sk-ant-xxxxx" | claude-keys store
```

### 4.5 验证认证

```bash
# 检查版本
claude --version

# 简单对话测试
claude -p "回复OK"
# 应该输出包含 "OK" 的回复

# 代码生成测试
claude -p "写一个python函数计算斐波那契数列第20项，直接输出代码不要解释" 2>&1 | tail -20
```

### 4.6 配置文件逐行解读

```json
{
  "model": "MiniMax-M2.7",
  // ↑ 使用的模型。M2.7 性价比高，日常使用推荐

  "permissionMode": "acceptEdits",
  // ↑ 权限模式：自动接受文件编辑，但执行命令仍需确认

  "env": {
    "ANTHROPIC_BASE_URL": "https://api.minimax.chat/anthropic",
    // ↑ API 地址。MiniMax 的 Anthropic 兼容接口
    //   也可能是 https://api.minimaxi.com/anthropic

    "ANTHROPIC_API_KEY": "sk-你的API-Key",
    // ↑ 你的 API 密钥（也可能是 ANTHROPIC_AUTH_TOKEN）

    "API_TIMEOUT_MS": "600000",
    // ↑ API 超时时间（毫秒）。600000 = 10 分钟。太短会频繁超时

    "DISABLE_NONESSENTIAL_TRAFFIC": "1",
    // ↑ 禁用遥测等非必要网络请求，保护隐私

    "CLAUDE_CODE_DISABLE_TERMINAL_TITLE": "1"
    // ↑ 禁止 Claude Code 修改终端标题
  },

  "autoCompact": {
    "enabled": true,
    "maxTokens": 512000
    // ↑ 当对话超过约 512K tokens 时自动压缩上下文
  }
}
```

## 第五章 三层配置架构

Claude Code 支持**三层配置**，后层覆盖前层。这让你可以在不同项目中灵活切换配置。

### 5.1 三层配置对比

| 层级 | 文件路径 | 作用范围 | 是否提交 Git | 典型用途 |
|------|----------|----------|:--:|------|
| **全局** | `~/.claude/settings.json` | 所有项目 | — | API 认证、默认模型 |
| **项目** | `<项目根>/.claude/settings.json` | 当前项目 | ✅ | 项目特定模型、权限 |
| **本地** | `<项目根>/.claude/settings.local.json` | 本机 + 当前项目 | ❌ | 个人 API Key、偏好 |

### 5.2 配置合并示例

假设你有以下配置：

**全局** `~/.claude/settings.json`：
```json
{ "model": "MiniMax-M2.7", "permissionMode": "default" }
```

**项目** `我的项目/.claude/settings.json`：
```json
{ "model": "MiniMax-M3" }
```

**最终生效**：
```json
{ "model": "MiniMax-M3", "permissionMode": "default" }
```

> 项目级的 `model` 覆盖了全局的 `model`，但全局的 `permissionMode` 保留。

### 5.3 安全建议

```bash
# 将本地配置文件加入 .gitignore
echo ".claude/settings.local.json" >> .gitignore
```

> ⚠️ `settings.local.json` 通常包含 API Key，**绝对不要**提交到 Git！

## 第六章 核心参数详解

### 6.1 完整参数列表

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `model` | string | — | 使用的模型名称 |
| `permissionMode` | string | `default` | 权限模式 |
| `autoCompact.enabled` | bool | true | 是否自动压缩长对话 |
| `autoCompact.maxTokens` | number | — | 触发压缩的 token 阈值 |
| `API_TIMEOUT_MS` | string | `600000` | API 请求超时（毫秒） |
| `DISABLE_NONESSENTIAL_TRAFFIC` | string | `0` | 禁用遥测流量 |
| `CLAUDE_CODE_DISABLE_TERMINAL_TITLE` | string | `0` | 禁用终端标题修改 |
| `CLAUDE_CODE_MAX_OUTPUT_TOKENS` | string | — | 最大输出 token 数 |

### 6.2 `model` — 模型选择

```json
"model": "MiniMax-M2.7"
```

- **M2.7**：均衡型，快且便宜，日常使用推荐
- **M3**：旗舰型，能力强但较慢，复杂任务推荐

### 6.3 `permissionMode` — 权限控制

详见[第八章](#第八章-权限模式)。

### 6.4 `autoCompact` — 自动压缩

Claude Code 对话越长，上下文窗口越容易爆满。`autoCompact` 自动压缩旧对话：

```json
"autoCompact": {
  "enabled": true,
  "maxTokens": 512000
}
```

- 当对话超过约 512K tokens 时，自动总结旧的对话内容
- 你也可以手动触发：`/compact`

### 6.5 `API_TIMEOUT_MS` — 超时设置

```json
"API_TIMEOUT_MS": "600000"
```

| 推荐值 | 说明 |
|--------|------|
| `300000` (5分钟) | 简单问答足够 |
| `600000` (10分钟) | **推荐**，适合代码生成 |
| `1800000` (30分钟) | 大型项目分析 |

> ⚠️ 不建议超过 30 分钟。如果频繁超时，检查网络或减小任务规模。

### 6.6 `DISABLE_NONESSENTIAL_TRAFFIC` — 隐私保护

```json
"DISABLE_NONESSENTIAL_TRAFFIC": "1"
```

设置 `"1"` 后，Claude Code 不会发送遥测数据、崩溃报告等非必要流量。

### 6.7 `CLAUDE_CODE_DISABLE_TERMINAL_TITLE` — 终端标题

```json
"CLAUDE_CODE_DISABLE_TERMINAL_TITLE": "1"
```

阻止 Claude Code 修改终端窗口标题。如果你使用 tmux 或自定义终端标题，建议开启。

## 第七章 模型选择策略

### 7.1 MiniMax 模型对比

| 维度 | MiniMax-M2.7 | MiniMax-M3 |
|------|:--:|:--:|
| **定位** | 均衡型 | 旗舰型 |
| **响应速度** | ⚡ 快 | 🐢 中等 |
| **复杂推理** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **代码质量** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **成本** | 💰 中 | 💰💰 高 |
| **上下文窗口** | 大 | 大 |

### 7.2 推荐策略

```
日常开发 / 代码解释 / 简单问答  →  MiniMax-M2.7
复杂架构 / 大型重构 / 算法设计  →  MiniMax-M3
单细胞数据分析 / 脚本编写      →  看任务复杂度
```

### 7.3 运行时切换模型

```bash
# 指定模型启动
claude --model MiniMax-M3

# 单次问答指定模型
claude -p "复杂分析问题" --model MiniMax-M3

# 交互式会话中切换
> /model
```

## 第八章 权限模式

### 8.1 四种模式对比

| 模式 | 读文件 | 编辑文件 | 执行命令 | 适用场景 |
|------|:--:|:--:|:--:|------|
| `default` | ✅ 自动 | ⚠️ 确认 | ⚠️ 确认 | 新手、生产环境 |
| `acceptEdits` | ✅ 自动 | ✅ 自动 | ⚠️ 确认 | **日常使用（推荐）** |
| `bypassPermissions` | ✅ 自动 | ✅ 自动 | ✅ 自动 | 完全信任环境 |
| `plan` | ✅ 只读 | ❌ 禁止 | ❌ 禁止 | 仅分析不出手 |

### 8.2 设置权限模式

```json
// 在 settings.json 中
{ "permissionMode": "acceptEdits" }
```

或在交互式会话中切换：

```
> /permission
```

### 8.3 安全建议

| 场景 | 推荐模式 |
|------|------|
| 个人开发环境 | `acceptEdits` |
| 生产服务器 | `default` |
| 代码审查 | `plan` |
| CI/CD 自动化 | `bypassPermissions`（配合沙箱） |

## 第九章 CLAUDE.md 记忆文件（⭐ 关键步骤）

> 🚨 **跳过此步骤会导致 Claude 不了解你的工作背景，回复质量大打折扣！**
>
> CLAUDE.md 是 Claude Code 的"长期记忆"。没有它，Claude 就像**一个不了解你的新同事**，每次对话都要从头说明你的工作背景和偏好。API 调用消耗的 token 会更多，因为你需要在每次对话中重复这些信息。
>
> 配置只需 **5 分钟**，但能显著提升后续所有对话的质量，并节省大量 token。

### 9.1 全局 CLAUDE.md（所有项目生效）

```bash
cat > ~/.claude/CLAUDE.md << 'EOF'
# 我的全局偏好

## 身份
- 生物信息学研究员，主要做单细胞转录组分析
- 工作环境：WSL2 Ubuntu 22.04

## 编程偏好
- Python 用 uv 管理依赖（`uv run`、`uv sync`）
- R 用 renv 管理包
- 代码注释用中文，变量名用英文
- Shell 脚本首行：`#!/bin/bash`

## 常用工具
- dnbc4tools：华大单细胞上游分析
- STARsolo：scRNA-seq 比对定量
- Seurat / Scanpy：下游分析

## 习惯
- 生成文件前先检查目录结构
- 不覆盖已有文件，除非我明确要求
- 给出命令时同时解释含义
EOF
```

### 9.2 项目级 CLAUDE.md

在项目根目录创建：

```bash
mkdir -p .claude
cat > .claude/CLAUDE.md << 'EOF'
# 项目特定说明

## 项目名称
dnbc4tools 单细胞分析项目

## 目录结构
- 上游/fastq/  -> FASTQ 原始数据
- 下游/0.RAW/   -> 分析输出
- Ref/          -> 参考基因组

## 关键约定
- 脚本放 scripts/ 目录
- 日志放 logs/ 目录
EOF
```

### 9.3 用 /init 自动生成

在 Claude Code 交互式会话中：

```
> /init
```

Claude 会自动分析当前项目，生成合适的 CLAUDE.md。

## 第十三章 FAQ 故障排查

### Q1: `claude: command not found`

```bash
# 检查是否安装
npm list -g @anthropic-ai/claude-code

# 添加到 PATH
export PATH="$HOME/.nvm/versions/node/$(node -v)/bin:$PATH"
echo 'export PATH="$HOME/.nvm/versions/node/$(node -v)/bin:$PATH"' >> ~/.bashrc
```

### Q2: `401 Unauthorized` 认证失败

```bash
# 检查 API Key 是否正确
cat ~/.claude/settings.json | grep -E "API_KEY|AUTH_TOKEN"

# ⚠️ MiniMax 不同环境可能使用不同字段名：
#   ANTHROPIC_API_KEY 或 ANTHROPIC_AUTH_TOKEN
# 如果一种不生效，请尝试另一种。详见第四章 4.3 节。
```

### Q3: `Request timed out` 请求超时

```bash
# 检查网络连通性
curl -I https://api.minimax.chat

# 增加超时时间
# 在 settings.json 中："API_TIMEOUT_MS": "1200000"
```

### Q4: `Model not found` 模型不存在

- 确认你在 MiniMax 平台上有该模型的访问权限
- 检查模型名拼写：`MiniMax-M2.7` 不是 `minimax-m2.7`
- MiniMax 的 Anthropic 兼容接口可能需要特定模型名

### Q5: Token 超出限制

```
> /compact
```

或降低 `autoCompact.maxTokens`：

```json
"autoCompact": { "maxTokens": 256000 }
```

### Q6: 权限确认太频繁

修改 settings.json：

```json
"permissionMode": "acceptEdits"
```

或在会话中切换：

```
> /permission acceptEdits
```

### Q7: npm 安装失败

```bash
# 清除缓存重试
npm cache clean --force
npm config set registry https://registry.npmjs.org/
npm install -g @anthropic-ai/claude-code
```

### Q8: Node.js 版本不兼容

```bash
# 确保使用 LTS 版本
nvm install --lts
nvm use --lts
nvm alias default lts/*
```

### Q9: 对话太长导致卡顿

```
> /compact
> /clear
```

建议定期 `/compact`，或设置更低的 `autoCompact.maxTokens`。

### Q10: 如何从 Anthropic 切换到 MiniMax？

编辑 `~/.claude/settings.json`：

```json
{
  "model": "MiniMax-M2.7",
  "env": {
    "ANTHROPIC_BASE_URL": "https://api.minimax.chat/anthropic",
    "ANTHROPIC_API_KEY": "sk-你的MiniMax-Key",
    "API_TIMEOUT_MS": "600000",
    "DISABLE_NONESSENTIAL_TRAFFIC": "1"
  }
}
```
